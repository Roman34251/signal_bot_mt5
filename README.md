# Signal Bot XAUUSD — MT5 + XGBoost

Бот генерує BUY/SELL сигнали по золоту (XAUUSD) на основі ML-моделі (XGBoost),
натренованої на реальних даних з MT5-термінала, і публікує їх у Telegram у
форматі: Entry-зона, Stop Loss, Take Profit 1-4.

## Архітектурне обмеження (важливо!)

Пакет `MetaTrader5` працює **тільки на Windows** і **тільки з локально
запущеним терміналом MT5** на тій самій машині. Тому:
- Бот і термінал MT5 повинні бути на одному Windows-сервері (VPS) або ПК.
- Лінукс-сервери для цього не підходять без Windows-емуляції.

## Крок 1 — підготовка Windows VPS

1. Орендуйте Windows VPS (форекс-орієнтовані хостери зазвичай зручніші:
   ForexVPS, Beeks, або звичайний Windows Server у будь-якого хмарного
   провайдера).
2. Підключіться через RDP (Remote Desktop).
3. Встановіть Python 3.10+ з python.org (позначте "Add to PATH" при
   встановленні).
4. Встановіть MT5-термінал (завантажте з сайту OANDA або
   https://www.metatrader5.com), увійдіть демо-рахунком OANDA
   (логін/пароль/сервер — з листа після реєстрації, сервер зазвичай
   `OANDA_Global-Demo-1`).
5. Переконайтесь, що золото додано в Market Watch термінала і **запам'ятайте
   точну назву символу** (може бути `XAUUSD`, `XAUUSD.m`, `XAUUSD.pro` залежно
   від брокера) — вона піде в `.env`.

## Крок 2 — встановлення бота

```powershell
# Розпакувати проєкт у, наприклад, C:\signal_bot
cd C:\signal_bot
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
notepad .env   # заповнити MT5_LOGIN, MT5_PASSWORD, MT5_SERVER, TELEGRAM_*
```

## Крок 3 — тренування моделі

```powershell
python train_model.py
```

Скрипт:
- підключається до MT5, тягне історію (роки задаються `MODEL_TRAIN_HISTORY_YEARS`)
- рахує walk-forward валідацію і друкує precision/recall по кожному фолду —
  **обов'язково подивіться на ці цифри перед тим, як довіряти сигналам**
- зберігає фінальну модель у `models/`

Якщо precision занадто низький (модель майже випадково вгадує) — підніміть
`MODEL_PROBABILITY_THRESHOLD` у `.env` (менше сигналів, але надійніші) або
збільшіть `MODEL_TRAIN_HISTORY_YEARS`, якщо термінал дозволяє більше історії.

## Крок 4 — запуск бота

```powershell
python main.py
```

Перевірте, що:
- в консолі з'являються рядки `P(buy)=... P(sell)=...` кожні
  `MT5_POLL_INTERVAL_SEC` секунд
- у Telegram прийшов heartbeat "✅ Бот працює"

## Крок 5 — робота 24/7 без відкритої консолі

Варіант А — Task Scheduler (простіше):
1. Планувальник завдань → Створити завдання
2. Тригер: "При запуску" / "При вході в систему"
3. Дія: запустити `C:\signal_bot\venv\Scripts\python.exe` з аргументом
   `C:\signal_bot\main.py`
4. У налаштуваннях: "Виконувати незалежно від того, чи ввійшов користувач"

Варіант Б — Windows-служба через NSSM (надійніше, автоперезапуск при краху):
1. Завантажте `nssm` (https://nssm.cc)
2. `nssm install SignalBot`
3. Вкажіть шлях до `python.exe` (з venv) і `main.py` як аргумент
4. `nssm start SignalBot`

**Обов'язково** увімкніть автологін термінала MT5 (Файл → зберегти сесію) і
автозапуск термінала при старті Windows, щоб після перезавантаження VPS все
піднімалось саме, без вашого втручання.

## Чесні обмеження поточної версії

- Rule-based/ML-точність **не гарантована** — ринок золота залежить від
  макроданих і новин, яких модель на цінових даних не бачить.
- `storage.update_outcome()` — заготовка для трекінгу фактичного результату
  сигналу (чи дійшла ціна до TP чи SL). У поточній версії це **не
  автоматизовано** — щоб реалізувати, потрібен окремий моніторинг ціни між
  сигналами, порівняння з рівнями і виклик `update_outcome()`. Без цього
  `get_accuracy_summary()` завжди буде порожнім.
- Cooldown і дедублікація — на рівні SQLite (переживає перезапуск, на
  відміну від попередньої email-версії).
- Forward-тест на демо перед довірою до сигналів — **обов'язковий**,
  мінімум кілька тижнів спостереження.
- Перенавчання моделі — раз на місяць-два рекомендовано (`python
  train_model.py` повторно), ринок змінює характер з часом.

## Структура проєкту

| Файл | Призначення |
|---|---|
| `config.py` | Всі налаштування з `.env` |
| `mt5_client.py` | Підключення до MT5 з auto-reconnect |
| `features.py` | Технічні індикатори (EMA, RSI, ATR, momentum) |
| `train_model.py` | Тренування XGBoost з triple-barrier розміткою і walk-forward валідацією |
| `ml_model.py` | Обгортка XGBoost (train/predict/save/load) |
| `levels.py` | Розрахунок Entry/SL/TP1-4 на основі ATR |
| `storage.py` | SQLite: історія сигналів, cooldown, heartbeat |
| `telegram_publisher.py` | Форматування і відправка в Telegram |
| `main.py` | Головний цикл: опитування → інференс → публікація |
